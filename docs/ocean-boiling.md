# 请不要煮沸的海水

> 原文：<https://web.archive.org/web/venturehacks.com/ocean-boiling>

[![](img/997bb150b45817577538f81704beda73.png)](https://web.archive.org/web/20220928212320/http://www.tbray.org/ongoing/When/201x/2010/01/02/Doing-It-Wrong)[【蒂姆·布雷】](https://web.archive.org/web/20220928212320/http://www.tbray.org/ongoing/When/201x/2010/01/02/Doing-It-Wrong)【XML 规范】合编:

> “如今的网络是这样的:从有一个想法到公开发布之间的时间是以天而不是月、周而不是年来计算的。每个后续发布周期都是如此。团队很小。进步是迭代的。*没有海洋是沸腾的，没有怪物要求写文档。*
> 
> “你得到了什么？脸书。谷歌。推特。拉韦利。大本营。TripIt。GitHub。等等等等。
> 
> “显然，技术很重要……更重要的是文化:迭代开发、持续重构、无处不在的单元测试、*从小处着手，在看起来合理之前收集用户体验*。”

斜体表示我的强调。阅读蒂姆·布雷的[做错事](https://web.archive.org/web/20220928212320/http://www.tbray.org/ongoing/When/201x/2010/01/02/Doing-It-Wrong)的其余部分。而我最喜欢的关于迭代软件开发的书是[极限编程讲解](https://web.archive.org/web/20220928212320/http://venturehacks.com/articles/extreme-programming-explained)，你可以在我们的[书店](https://web.archive.org/web/20220928212320/http://venturehacks.com/bookstore)找到。
